WebShield

A rule-based phishing detection system that analyzes URLs and classifies them as Safe, Suspicious, or Phishing.

Problem Statement

Phishing attacks have become one of the most common cybersecurity threats, where attackers create fake websites that closely resemble legitimate ones to steal sensitive user information. Traditional detection methods such as blacklist-based systems fail to detect newly generated phishing URLs, while machine learning approaches require large datasets and computational resources. There is a need for a lightweight, fast, and explainable system that can detect phishing links in real time without complex dependencies. This project aims to address this problem by providing an efficient rule-based URL analysis system.

Literature Review

Phishing detection has been widely studied using both blacklist-based and machine learning approaches. Traditional methods like Google Safe Browsing rely on databases of known malicious URLs, but they fail to detect new or zero-day attacks. Machine learning models improve detection but require datasets, training, and computational power.

Research Gap

Most existing phishing detection systems rely on blacklist databases or machine learning models. While blacklist-based approaches fail to detect newly generated phishing websites, machine learning methods require large datasets and training time. Additionally, many ML-based systems lack interpretability, making it difficult to understand why a URL is malicious. This project addresses these limitations by introducing a lightweight rule-based system that works in real time without requiring training data. It provides faster results and clear reasoning behind each classification.

System Methodology

Input

User enters URL → system analyzes instantly.

Architecture

1. URL Parsing: Extract domain, protocol, and structure.

2. Feature Extraction:
• URL length
• Number of dots
• Special characters (@, -, _)
• HTTPS usage
• Suspicious keywords

3. Scoring Mechanism: Assign weights to each feature and calculate total risk score.

4. Classification: Safe / Suspicious / Phishing based on score.

Rules Used

  • IP Address → suspicious if used
  • @ Symbol → misleading
  • Punycode → encoded domains
  • Suspicious TLD → .xyz, .tk
  • Keywords → login, verify
  • Too many dots → fake structure
  • Long URL → hides intent
  • Hyphen → fake domain
  • No HTTPS → insecure

Scoring System

  • IP Address → +25
  • @ Symbol → +25
  • Punycode → +25
  • TLD → +15
  • Keywords → +10
  • Dots → +8
  • Length → +8
  • Hyphen → +8
  • No HTTPS → +10

Safe: 0–29 | Suspicious: 30–59 | Phishing: 60+
