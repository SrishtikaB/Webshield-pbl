WebShield

A rule-based phishing detection system that analyzes URLs using multiple security indicators and classifies them as Safe, Suspicious, or Phishing.

URL Input

The user enters a URL into the system for analysis. The rule-based engine evaluates the link using multiple phishing indicators and assigns a risk score to classify the URL.

Problem Statement

Phishing attacks are one of the most common cybersecurity threats. Users often cannot distinguish legitimate and malicious URLs. This project proposes a lightweight rule-based detection system.

Research Gap

Most phishing detection systems rely on machine learning models that act as black boxes. This project focuses on explainability, transparency, and rule-based scoring.

Results & Analysis

The system was tested on 35 URLs including legitimate and phishing links. 31 URLs were correctly classified, resulting in 88% accuracy. The model effectively identified IP-based URLs, suspicious keywords, punycode domains, and brand impersonation patterns.

88% Accuracy

Most phishing URLs triggered multiple detection rules. The most common indicators were suspicious keywords and unusual domain extensions. Safe URLs typically scored below 20.

System Interpretation

  • Each triggered rule adds a specific score.
  • The final classification is transparent and explainable.
  • No black-box decision making.
  • Users can see exact reasons for detection.

Limitations

  • No real-time API integration.
  • Cannot detect newly registered domains.
  • May generate false positives.
