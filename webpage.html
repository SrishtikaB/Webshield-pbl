WebShield: Phishing URL Detection System

"A lightweight rule-based cybersecurity system designed to detect phishing URLs in real-time by analyzing structural and behavioral features of web links, improving user safety and preventing credential theft."

Problem Statement

Phishing attacks are one of the most widespread cyber threats where attackers create fake websites that mimic legitimate platforms to steal sensitive information such as passwords, credit card details, and personal data. Users often fail to distinguish between genuine and malicious URLs due to their similar appearance. Existing solutions either rely heavily on machine learning models or browser-based detection, which may not always be fast or accessible. Hence, there is a need for a simple, fast, and efficient system that can detect phishing links in real time without heavy computation.

Literature Review / Market Research

Existing research shows that phishing detection systems commonly use machine learning algorithms such as Decision Trees, Random Forest, and Neural Networks. These systems analyze features like URL length, domain age, HTTPS usage, and suspicious keywords. However, they require large datasets, training time, and computational power. Popular tools like Google Safe Browsing and antivirus software also detect phishing but are not always real-time or transparent.

Research Gap / Innovation

Most existing systems are complex and resource-intensive. Our approach introduces a rule-based system that eliminates the need for training datasets while still achieving high accuracy. It is faster, explainable, lightweight, and suitable for real-time applications such as browser extensions or small-scale security tools.

System Methodology

Dataset / Input

A total of 300 URLs were tested:
• 100 Legitimate URLs
• 100 Phishing URLs
• 100 Suspicious URLs The URLs were collected from public datasets and manually verified. Preprocessing included removing duplicates and normalizing URL formats.

Model / Architecture

The system uses a rule-based scoring mechanism:
• Checks URL length
• Detects special characters (@, -, _)
• Verifies HTTPS usage
• Identifies suspicious keywords Each feature contributes to a score, and the final classification is: Safe / Suspicious / Phishing.
